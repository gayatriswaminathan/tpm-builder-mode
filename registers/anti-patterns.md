# Anti-Pattern Register

> Pattern **#4**: a living "things that have bitten us" library, keyed by work-type, **re-read before related work** — a pre-mortem fed by real postmortems. Each entry names the dated wound, why it happened, and the guard that prevents recurrence.

How to use: before a deploy / migration / vendor cutover / launch, read the entries tagged for that work-type. Add a new entry whenever something bites.
